# BSA2_Requirements

Проект по анализу требований и моделированию процессов AS IS / TO BE для двух кейсов: онлайн-запись в барбершоп (BRB) и сервис доставки (DLV).

## 🛠 Навыки
- Requirements Analysis
- AS IS / TO BE Modeling
- Context Diagram
- Data Streams Analysis
- Glossary Management

## 📋 Описание проекта
В ходе работы я провела анализ текущего состояния процессов, выявила проблемы и предложила решения для двух сервисов: барбершоп (онлайн-запись) и доставка (курьерская служба).

### Что именно было сделано:
1. **Модель AS IS (BRB)**: Выявила действия и проблемы стейкхолдеров в текущих процессах (по телефону и очно).
2. **Модель AS IS (DLV)**: Выявила действия и проблемы стейкхолдеров в текущих процессах.
3. **Контекстная диаграмма (DLV)**: Построила диаграмму взаимодействия системы с внешними сущностями.
4. **Модель TO BE (BRB)**: Разработала целевые действия и решения для стейкхолдеров.
5. **Модель TO BE (DLV)**: Разработала целевые действия и решения для стейкхолдеров.
6. **Потоки данных (BRB)**: Составила таблицу входов/выходов для стейкхолдеров.
7. **Потоки данных (DLV)**: Составила таблицу входов/выходов для стейкхолдеров.
8. **Глоссарий**: Расширила понятийный аппарат для BRB и DLV.
9. **Каталог стейкхолдеров**: Актуализировала каталог для BRB и DLV.
